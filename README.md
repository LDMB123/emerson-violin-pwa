# Emerson Violin PWA

Repository for Emerson's Violin Studio, a local-first violin practice app built for iPad-first lessons, guided home practice, and offline use.

Installed app metadata comes from [public/manifest.webmanifest](public/manifest.webmanifest):

- `name`: `Emerson's Violin Studio`
- `short_name`: `ViolinPanda`

## What Ships In The App

- Mission-driven home and coach flows for daily violin practice
- Practice tools including tuner, metronome, drone, bowing, and posture helpers
- Songs plus interactive games for pitch, rhythm, memory, and technique work
- Parent-zone surfaces for goals, recordings, review, settings, and data controls
- Local-first persistence with installable PWA behavior, shortcuts, and offline recovery

## Health Check

Avoid relying on dated snapshots in docs. Verify repo health from the current checkout:

- `npm run handoff:status`
- `npm run handoff:verify`

## Stack

- Vite + React 19 + React Router 7
- Vitest + Playwright
- GitHub Pages SPA fallback generated by `scripts/build-spa-fallback.mjs` during `postbuild`
- Service worker precache list generated by `scripts/build-sw-assets.js`
- IndexedDB-first persistence with localStorage fallback
- Optional WASM modules under `src/wasm/` and `wasm/`

## Deployment Shape

- Static app build produced by `npm run build`
- SPA fallback emitted as `dist/404.html` for GitHub Pages-style hosting
- Service worker asset manifests generated before dev/build and refreshed after production builds
- Installed app entrypoint and shortcuts defined in [public/manifest.webmanifest](public/manifest.webmanifest)

## Quality Gates

- `audit:docs` is the fast freshness guard for live Markdown and exported/shared JSDoc contracts.
- `audit:static` is the zero-dependency audit lane: secrets, docs freshness, duplicate-dependency allowlist check, perf budget config, home view sync, feature module completeness, accessibility, and learning completeness.
- `audit:dep-backed` is the dependency-backed verification lane: lint, duplicate clone scan, dead-code scan, coverage effectiveness, and production build.
- Dead code and unused exports are checked by `knip` using `knip.json`.
- Duplicate dependency versions are checked by `scripts/audit-dependency-duplicates.mjs`.
- Secret/credential pattern leakage is checked by `scripts/audit-secrets.mjs`.
- Performance budget config drift is checked by `scripts/audit-performance-budget-config.mjs`.
- `audit:full` runs `audit:static` plus `audit:dep-backed`.
- CI runs the static audits, dependency-backed verification, production dependency vulnerability audit, and Playwright E2E on PRs and pushes to `main`.
- Operator runbook for zero-context pickup: `docs/HANDOFF.md`.

## Architecture Notes

- `index.html` provides document metadata, global dialogs, and the `#react-root` mount point; `src/app.jsx` boots the React app.
- `src/AppShell.jsx` owns the top bar, bottom nav, `#main-content` frame, legacy hash compatibility bridge, and route-level `Suspense` boundaries.
- Pathname routing lives in `src/routes.jsx` via `createBrowserRouter()`.
- Most user-facing surfaces now live in native React views under `src/views/`.
- Legacy HTML/WASM songs and games still ship under `public/views/` and are hosted inside React by `src/views/Songs/SongRunnerView.jsx` and `src/views/Games/GameRunnerView.jsx`.
- `src/app/module-registry.js` and `src/app/legacy-view-runtime.js` still hydrate the remaining DOM-bound helper modules for legacy view ids where behavior parity matters.
- Low-priority runtime preloads still use the idle-task path with `requestIdleCallback` fallback where available.
- Shared audio element rewriting still flows through `prepareAudioElementSource()` for loaded legacy markup.
- Song playback and song recording are intentionally different routes: normal play should work without microphone permission, record intent should request it.
- Recording persistence is blob-first; UI playback should resolve sources through the persistence helpers rather than reading `dataUrl` directly.
- Home route entry point: `src/views/Home/HomeView.jsx`.
- Live subsystem docs live under `docs/architecture/`; avoid duplicating fast-moving feature inventories in this file.
